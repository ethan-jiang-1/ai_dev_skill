# round2_src Wave 0 权威来源优先级清单

目标：先建立 4 个主题共用的 high-trust ground truth 地基。这里先列“应该优先抓的权威来源类型与对象”，不在此文件里写结论。

## 共享（Wave 0）

优先抓：

- AgentSkills / `SKILL.md` 规范的官方页面与规范文本（定义、目录结构、加载/发现规则、渐进式披露）
- 主要宿主平台的官方文档（skills/rules/workflows/plugins/subagents 的机制与目录约定）
- 主要分发与聚合基础设施的官方仓库与文档（安装模型、兼容策略、版本/安全治理）
- 至少 1 份安全研究或协议/供应链分析（prompt injection、静默更新、脚本执行风险、签名/验证/不可变性）
- 至少 1 份高质量对比或实践复盘（官方机制 vs 社区实践、工具之间差异）

落地方式：每份来源沉淀为 `reference_src/00-shared-*.md`，并在 `Claims Supported` 写清楚它支撑哪些判断。

## 主题 1：基础规范与宿主平台生态（host）

优先抓官方材料：

- `SKILL.md` / AgentSkills 规范文本与变更记录（如有）
- Claude Code / Codex / Cursor / Windsurf / OpenCode / Copilot 等宿主的官方文档：skill 的发现、加载、触发、权限、组合
- 宿主平台的“参考实现”仓库（如官方 skills 示例库、官方工具/脚手架）

## 主题 2：聚合器、注册表与分发安全（dist）

优先抓官方材料：

- 典型安装器/CLI/registry 的官方仓库与文档（安装模型、兼容探测、目录映射）
- 安全与版本治理平台的机制文档（语义化版本、不可变性、验证/扫描流程）
- 如果存在：安全扫描器/检查器的官方仓库或白皮书

## 主题 3：企业官方来源、社区索引与 MCP 共生（supply）

优先抓官方材料：

- 企业第一方 skill/agent 相关仓库（与自家产品版本同步的示例与 SOP）
- Awesome/目录类索引的“入选标准”与维护机制（如有）
- MCP 生态的官方协议、目录、检查器/验证器文档（边界、互操作、组合方式）

## 主题 4：SDLC 方法论框架与工程治理（framework）

优先抓官方材料：

- 各框架的官方仓库 README、docs、发布说明（机制、流程、产物、约束点）
- 框架的“关键机制”证据：如何防漂移/防漏验证/防跑偏、如何治理状态、如何 gate

同时补一类对照材料：

- 社区真实使用反馈与反例（失败模式、复杂度、迁移成本、维护负担）
