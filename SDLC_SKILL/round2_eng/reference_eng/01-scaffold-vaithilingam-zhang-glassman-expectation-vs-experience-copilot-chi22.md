## 关键事实

- 研究设计（摘要/方法）：
  - within-subjects user study，n=24。
  - 对照条件：VSCode + Intellisense（默认 code completion）。
  - 实验条件：VSCode + Copilot（可通过注释作为 prompt）。
  - 任务：3 个真实 Python 任务（easy/medium/hard），每个任务 20 分钟上限；允许互联网搜索（论文写明允许使用 online resources）。
- 主要结果（摘要）：
  - Copilot 不一定提升完成时间或成功率；
  - 但多数参与者更愿意在日常工作中使用它，因为它给出有用起点，减少 online search effort；
  - 同时，理解/编辑/调试生成片段的困难显著，影响任务解决效果。

## 与本研究的关系

- 对 01-scaffold：
  - 支撑一个关键事实：AI code generation 的净收益不是单调的，可能出现“偏好高但效果未必更好”的现象；主要障碍集中在理解与修复，而这正是脚手架应介入的环节（verification/repair）。
  - “减少 online search effort”是典型 offloading 方向信号；结合理解/调试困难，可以用于讨论 cognitive debt 风险（但需要谨慎，论文本身不直接用该术语）。
- 对 Skill 设计：
  - 暗示一个可操作方向：把“生成后的理解与修复”当作一等公民设计（例如：强制解释、要求最小可验证例子、引导定位错误与回归测试），而不是只优化生成质量。
