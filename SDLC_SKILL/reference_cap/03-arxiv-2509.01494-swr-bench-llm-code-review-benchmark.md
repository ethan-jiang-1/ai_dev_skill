## 关键事实

- 提出 SWR-Bench：1000 个来自 GitHub 的真实 PR（manually verified），强调 PR-centric review 与“完整项目上下文”。
- 提出一种 objective 的 LLM-based evaluation：通过验证生成 review 是否覆盖 structured ground truth 的 issues，并在摘要中宣称与 human judgment 强一致（约 90% agreement）。
- 系统性评估主流 ACR tools/LLMs 后指出当前系统整体 underperform，且更擅长发现 functional errors。
- 提出并验证 multi-review aggregation：对多份 review 的点进行分析/过滤/丢弃错误建议，综合成更好的 final report；摘要中给出 F1 up to +43.67% 的提升幅度。

## 与本研究的关系

- 支撑 `digested_cap/03` 的关键机制判断：review/critic/consensus 的价值不仅是“更多意见”，而是通过聚合与筛选提升覆盖与可靠性。
- 为 gstack 的“adversarial passes + cross-model synthesis”提供独立第三方基准证据与改进方向（多 reviewer 聚合）。

## 风险与局限

- 摘要中的“~90% agreement”依赖其评价方法与标注/ground truth 构造方式；落地时仍需结合组织内代码库与风险面复核。
- “聚合策略有效”不等于“无门禁即可自动合并”；仍需与测试、发布与监控等 ops 能力单元形成闭环。
