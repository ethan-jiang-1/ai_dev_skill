## 关键事实

- 摘要指出：自动修复若仅基于静态工件，会错过诊断复杂故障所必需的动态执行上下文。
- 提出 DebugHarness：围绕可复现 crash，采用 pattern-guided 调查策略，形成假设，并交互式 probe 内存状态与执行路径，然后闭环验证生成的 patch。
- 文中通过示例对比 “Static Agent vs Interactive Debugging”，展示静态 agent 易停留在 symptom site，而交互调试可通过 watchpoints 追溯到更早的 root cause（例如 cache invalidation 缺失）。
- 在真实 C/C++ 漏洞数据集 SEC-bench 上评估（摘要级描述）。
