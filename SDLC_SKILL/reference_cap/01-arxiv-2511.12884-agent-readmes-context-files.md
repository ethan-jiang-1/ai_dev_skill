## 关键事实

- 研究对象：2,303 个 agent context files，来自 1,925 个 repositories。
- 论文结论（摘要级）：
  - 这些文件并非静态文档，而是复杂、难读的工件，像配置代码一样演化，并以频繁、小幅添加为主。
  - 开发者写得最多的是功能性指令（摘要给出比例：Build/Run 62.3%、Implementation Details 69.9%、Architecture 67.7%）。
  - 非功能要求显著不足（Security 14.5%、Performance 14.5%）。
- 含义：context files 可以让 agent “能跑起来”，但不足以保证“安全/性能/质量”。

## 与本研究的关系

- 支撑 digested_cap/01：context files 的现实内容结构说明“计划与上下文”更多解决功能性可执行性，但需要 hooks/gates/验证前置来补足非功能约束。
- 支撑 digested_cap/03/04：企业迁移必须把安全/性能/合规纳入确定性 gate，否则上下文文件很可能缺失这些约束。
