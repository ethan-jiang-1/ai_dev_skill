## 关键事实

- 研究背景：静态分析工具在大规模工程中常有极高假阳性率，导致大量人工 inspection 成本与流程效率问题。
- 研究场景：腾讯（Tencent）真实工业流水线，使用企业定制 SAT（论文描述为 BkCheck），并集成到 code review/check-in 流程。
- 数据集切片：433 个 alarms（328 false positives / 105 true positives），覆盖 3 类常见 bug（如 NPD/OOB/DBZ），来自腾讯广告与营销服务（AMS）业务线相关仓库（论文描述的数据范围为 2024-09 至 2025-06）。
- 人工成本：论文在摘要与研究设置中给出“单 alarm 人工审查需要 10-20 分钟量级”的结论口径（并提及访谈/问卷发现）。
- 效果结论（摘要口径）：LLM 与静态分析结合的 hybrid 技术可消除 94%-98% 的 false positives，并保持高 recall（论文用语为 “high recall”）。
- 成本估计（摘要口径）：LLM-based techniques per-alarm 平均时间成本约 2.1-109.5 秒、平均 API 花费约 $0.0011-$0.12。

## 与本研究的关系

- 用于 `round2_cap/03`（审查/质量门禁）回答“企业里 reviewer/门禁最贵的是什么”：
  - 假阳性本质上是“门禁噪音”，吞噬 reviewer 时间并降低 gate 的可信度。
  - LLM 的价值不只在生成 patch，而是在“triage/过滤/解释”的 review 子任务上可形成可度量的质量单元（capability unit）。
- 用于 `round2_cap/04`（迁移价值）补齐“correctness/质量净收益”证据：
  - 不是只谈效率（runtime/tokens），而是谈企业质量流程中的真实节省（人力分钟/告警噪音/门禁可信度）。

## 风险与局限

- 研究对象是单一企业/单一业务线/单一内部 SAT 的真实数据切片，外推到不同公司与不同静态分析器需要谨慎。
- 论文覆盖的 bug 类型有限（3 类），且强调高 recall 约束；不同组织对漏报容忍度不同。
- 成本数据依赖当时的模型/API 价格与 prompt/方法实现；需要按企业实际运行条件重估。
