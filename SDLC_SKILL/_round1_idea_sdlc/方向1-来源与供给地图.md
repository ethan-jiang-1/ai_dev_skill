# 方向1：来源与供给地图

## 任务

这份 Deep Research 只研究一件事：

**公开生态里，哪些地方最可能持续产出偏软件开发的 skill、workflow、plugin、command、subagent？**

目标不是深挖单个 skill，  
而是先找到值得长期追踪的来源和高密度入口。

## 研究边界

这里要找的不是普通 prompt 仓库，  
而是会影响 coding agent 做事方式的结构化能力包。

优先关注：

- coding agent、AI IDE、CLI agent 的官方文档、examples、templates、marketplace
- GitHub 上面向开发流程的 skill / workflow / plugin / subagent 仓库
- 组织仓库、starter repo、awesome list、registry、catalog
- 社区里反复出现、并且能指向实际代码或安装入口的资源

默认弱化：

- 只有 prompt 文本、没有结构化能力单元的仓库
- 与软件开发无关的泛 AI 工具
- 只讲理念、不指向实际产物的文章

下面这些名字可以当作边界样本：

- get-shit-done
- gstack
- Google agent skills
- superpowers

## 要看什么

对每个来源，重点判断：

- 这里是否稳定产出 SDLC 相关能力包
- 更偏官方、社区，还是个人实验
- 可见度和活跃度怎么样
- 是否容易继续扩展出更多候选对象
- 是否偏向软件开发而不是泛效率工具

## 交付

最终输出尽量包含：

- 一张来源地图
- 一批值得继续深挖的来源
- 每个来源下的代表性项目或入口
- 对来源价值的简短判断

如果需要方便后续合并，可选记录这些最小字段：

- `name`
- `url`
- `source_type`
- `host_ecosystem`
- `notes`

## 搜索方向

优先沿这些方向找：

- 宿主官方入口：Claude Code、Codex、Cursor、Windsurf、OpenCode 的 docs、examples、templates、plugins、skills
- GitHub 入口：organization、template、starter、awesome、marketplace、registry、catalog
- 关键词：`coding agent skills`、`agent workflow coding`、`developer subagent`、`AI IDE plugins workflow`
- 结构线索：`SKILL.md`、`hooks`、`subagents`、`commands`、`workflow`、`planner`、`reviewer`

研究时宁可先找到高密度来源，  
也不要一开始就陷进单个仓库细节里。
