## 关键事实

- 论文对比多种 program repair 范式：static debugging、static simulation debugging、log augmented debugging，并指出它们的两类根本限制：
  - 对 runtime behavior 理解不足（LLM 模拟 execution trace 可能幻觉，verbose logs 可能淹没信号）
  - 缺少增量调试验证（只看最终 test outcome，过程奖励缺失）
- 提出 InspectCoder：dual-agent（Program Inspector + Patch Coder），Inspector 通过 debugger 做断点与状态检查，Coder 基于 root cause 产出 patch 并跑测试验证。
- 提供 InspectWare middleware，维护 stateful debugging sessions 以支持多轮交互。
- 在两个 self-repair benchmarks 上报告相对 baseline 的 repair accuracy 与效率提升。
