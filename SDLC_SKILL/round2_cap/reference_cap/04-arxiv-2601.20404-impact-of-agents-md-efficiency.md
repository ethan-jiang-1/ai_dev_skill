## 关键事实

- 研究对象：评估 AGENTS.md 对 AI coding agent（文中选择 OpenAI Codex）在 GitHub PR 任务上的效率影响。
- 样本与设计（摘要与方法段）：
  - 分析 10 个仓库、124 个 PR
  - paired within-task 设计：同一任务在 “有/无 AGENTS.md” 两种条件下运行，控制仓库/任务/代码状态，仅改变 instruction 文件是否存在
  - 在隔离 Docker 环境中重复执行，每个任务从相同 repo snapshot 起步，避免状态复用造成混淆
- 主要结果（表格与结果段）：
  - wall-clock median 从 98.57s 降到 70.34s（约 -28.64%，p<0.05）
  - output tokens median 从 2,925 降到 2,440（约 -16.58%，p<0.05）
  - mean 也下降（幅度与 median 接近），作者解释为整体分布左移而非少量极端值驱动
- 作者强调：这是一项效率维度的初步研究， correctness/quality 的系统评估属于后续研究路线。

## 与本研究的关系

- 为 `round2_cap/04` 的“迁移价值”提供直接证据：AGENTS.md 这类仓库级工件不仅是‘口径’，在可控实验中与效率提升相关，具有较高的可迁移价值与可投资性（前提是能控制内容质量与漂移）。
- 也提示了迁移边界：效率提升不等价于质量提升；企业落地需配套 review/verify/ops 的门禁链条（03 维度）来保证 correctness。
