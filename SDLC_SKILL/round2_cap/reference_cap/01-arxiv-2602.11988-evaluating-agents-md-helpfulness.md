## 关键事实

- 论文研究问题：仓库级 context files（如 AGENTS.md）是否真的能提升 coding agents 的真实任务表现。
- 摘要级主要结论（节选）：
  - 在多个 agents 与 LLMs 上，context files 倾向于降低任务成功率（相对“不给 repo context”），同时把 inference cost 提高 20%+。
  - 行为上，context files 会鼓励更广探索（更彻底的测试与文件遍历），且 agents 倾向于遵守其中指令。
  - 结论：不必要的要求会让任务更难；人写 context files 应描述“最小要求”。
- 论文贡献包括一个用于评估 context files 影响的基准（AGENT BENCH）。

## 与本研究的关系

- 直接支撑 cap plan 中“必须覆盖反例/局限/失败模式”的要求：planning 机制也可能伤害成功率与成本。
- 为企业迁移提供约束：AGENTS.md/CLAUDE.md 等 context file 不应越写越长，应与“最小 requirements + 机器可验证 gates”配合，否则容易 Goodhart（把 agent 绑死在不必要流程上）。

## 风险与局限

- 结果依赖实验设置（agents、LLMs、context file 生成方式、任务类型）；迁移到企业代码库前仍需做本地 A/B 验证。
- “最小 requirements”如何界定需要与组织的合规/安全/质量要求共同约束，不能为了成功率牺牲必要的安全门禁。
