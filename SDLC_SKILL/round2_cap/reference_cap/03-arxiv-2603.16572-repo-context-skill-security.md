## 关键事实

- 论文指出技能市场出现“类似移动 app store”的 skills marketplace，并伴随自动化 skill scanner。
- 摘要报告：
  - 收集 238,180 个 unique skills（来自三个主要平台与 GitHub）
  - 仅分析技能描述（SKILL.md）会导致高比例被标成 malicious（市场给出的比例最高可到 46.8%）
  - 引入“仓库上下文”后，非良性标记比例显著下降（摘要中给出降至 0.52% 的量级）
  - 揭示新的攻击向量：劫持被弃置的 GitHub 仓库并影响被索引的 skills

## 与本研究的关系

- 为 round2_cap/03/04 提供“技能作为供应链”的安全证据：企业引入 skills 生态必须做 provenance、仓库上下文校验与生命周期治理，而不是只看 SKILL.md 文本或 marketplace 标记。
- 对能力单元架构：说明“Skill-like units”如果可执行/可安装，就进入供应链风险域，需要 hook/gate 与审计。
