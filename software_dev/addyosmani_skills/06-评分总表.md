# Agent Skills 评分总表

## 评分维度说明

- 结构化程度：流程是否清楚、是否容易标准化执行
- 验证强度：是否明确要求测试、证据、运行态验证或检查点
- 复用范围：不同项目和不同任务中是否高频可复用
- 工具依赖：对外部平台、基础设施、浏览器、CI、监控的依赖程度
- 协作价值：对多人协作、跨角色协作、长期维护的帮助

分值范围：1 到 5，越高越强。

## 最依赖组织基础设施的 skill

| Skill | 为什么依赖高 |
|---|---|
| browser-testing-with-devtools | 需要浏览器与 DevTools 运行环境 |
| ci-cd-and-automation | 需要 CI 平台、脚本、质量门禁体系 |
| performance-optimization | 需要 profiling、监控、trace 或预算机制 |
| shipping-and-launch | 需要 feature flag、监控、回滚流程 |

## 最适合小团队优先采用的 skill

| Skill | 原因 |
|---|---|
| incremental-implementation | 低成本高收益 |
| test-driven-development | 很快提升可靠性 |
| code-review-and-quality | 能快速改善交付质量 |
| git-workflow-and-versioning | 直接改善提交历史与回滚能力 |
| context-engineering | 对 AI 协作效果立竿见影 |

## 最容易流于形式的 skill

| Skill | 风险 |
|---|---|
| spec-driven-development | 写模板不写真约束 |
| planning-and-task-breakdown | 机械拆任务，不改善执行质量 |
| documentation-and-adrs | 只记录结论，不记录 trade-off |
| performance-optimization | 不测量就“优化” |
| shipping-and-launch | 没有监控与灰度时只剩 checklist |
