# superpowers 总览

## 这是个什么项目

superpowers 是一个面向 coding agent 的技能工作流插件。  
它不是单一 skill，也不是单一宿主插件，而是：

- 一组显式 `SKILL.md`
- 一层会话启动注入
- 一组跨宿主安装与适配说明
- 一套验证这些 skill 是否真的被执行的测试

从 `README.md` 的表述看，它主打的是完整软件开发流程：

- brainstorming
- using-git-worktrees
- writing-plans
- subagent-driven-development / executing-plans
- test-driven-development
- requesting-code-review
- finishing-a-development-branch

## 项目主轴

superpowers 的主轴很清楚：  
不是尽量多做技能种类，而是尽量把 agent 推进到一条强纪律的软件开发流程里。

可以把它概括为：

- 启动时先注入“如何使用技能”
- 每次任务先判断该用哪一个 skill
- 然后沿着 spec -> plan -> execute -> review -> finish 的链路推进
- 再通过行为测试验证这套链路真的发生

## 它和普通 prompt 仓库的差别

普通 prompt 仓库常见形态是：

- 若干 Markdown 文件
- 各自描述一个角色或技巧
- 用户手动挑着用

superpowers 的差别在于：

- 它把 `using-superpowers` 当 bootstrap
- 它用 hook 主动把规则塞进 session
- 它有明确的 skill 触发纪律
- 它有集成测试去验证 skill 是否真的被调用

所以它更像 **workflow-enforcing skill plugin**，  
而不是“技能文档集合”。

## 总览结论

如果先不急着评分，只看总览层，  
superpowers 最值得注意的特点是三点：

- skill 数量不算夸张，但流程编排非常强
- 自动注入和宿主适配比很多 skill 仓库更成熟
- 测试意识很强，已经开始验证真实 agent 行为
