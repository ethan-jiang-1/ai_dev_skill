## 关键事实

- GitHub Changelog 的标题直接是 `GitHub Copilot now supports Agent Skills`。
- 页面正文说明，用户已经可以创建 Agent Skills，让 Copilot 学会以更稳定、可复用的方式完成特定任务。
- 页面再次把 Agent Skills 描述为由 instructions、scripts、resources 组成的文件夹，并强调它会在相关时自动加载。
- 页面还指出，如果把 skills 放进仓库的 `.claude/skills` 目录，Copilot 也能识别并使用。

## 与本研究的关系

- 对 `03` 而言，这是一条很强的生态信号：主平台不只是允许 skill 存在，而是正式把它作为能力扩展方式推向用户。
- 这也支撑一个更现实的判断：去网络上找现成 skill、观察他人如何组织 skill，不是旁门左道，而是顺着主平台正在强化的方向在学习。
- 对最终推荐来说，凡是能接到 GitHub 主平台 skill 语境的对象，跟踪价值都高于纯概念型仓库。

## 风险与局限

- 这条来源证明的是 GitHub 产品层支持与认可，不等于所有第三方 skill 仓库都已经成熟可靠。
- 该页面篇幅很短，更像能力发布信号，不提供细粒度采用统计。
