## 关键事实

- Vercel 在文中直接把 skills 形容为 `the "npm moment" for context`。
- 文中把 `Progressive disclosure` 作为设计逻辑之一，强调 skill 应在相关时才介入。
- 文中明确提出 `description is the trigger`，说明 skill 的描述质量直接影响能否被正确调用。
- 文中专门比较了 `Skills vs AGENTS.md`，说明生态已经在帮助使用者区分 always-on 仓库说明与按需 skill。
- 文中还明确提醒使用者要 `Treat skills like code`，并且 `Read them before installing`。
- 文中提到 `There’s no special publish command for skills.sh` 与 `install telemetry`，说明共享、安装和信号回流已经被设计进生态。

## 与本研究的关系

- 这份材料最直接支撑用户提出的成长命题: 现成 skill 通常比从零闭门琢磨更高杠杆，因为它们天然提供结构样板、触发方式和真实任务切片。
- 同时，它也提醒研究不能把“好找”误判成“可以不看就装”，否则会把学习入口和安全边界混掉。
- 对 `03` 来说，这份 guide 让我们可以更清楚地区分:
  - 学习杠杆
  - 分发便利
  - 工程可信度

## 风险与局限

- 这份来源来自同一生态推动方，对 skills.sh 路线天然更熟悉也更友好。
- 它更适合解释机制和实践建议，不提供独立第三方采用数据。
