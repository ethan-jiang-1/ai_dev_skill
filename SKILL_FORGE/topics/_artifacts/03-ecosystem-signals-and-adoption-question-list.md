# 03 / 生态信号、可信度与采用判断 / Question List

- `status`: `in_progress`

## 已初步回答

- 哪些外部 skill 资源对成长最有学习杠杆
  - 目前最强的是 `skills.sh` 目录层、`github/awesome-copilot` 聚合层、`vercel-labs/agent-skills` 官方样板层。
- 哪些采用信号足以支撑持续跟踪，而不是只算热闹
  - 目前已确认的强信号包括: 官方产品发布、官方仓库近期提交、分发层下载量、目录站公开统计、治理动作。
- 目录站、社区聚合和真实工程基座之间的关系应如何判断
  - 目前可以先分成 `directory / marketplace`、`community hub`、`official sample library`、`toolchain / governance` 四类对象。

## 仍待补充

- 哪些非官方第三方 skill 仓库已经表现出持续复用，而不是一次性热点
- 目录内 skill 的质量分布大致如何，是否存在明显长尾污染
- 有没有更具体的外部事故、权限误用或恶意 skill 案例可作为失败样本
- 最终推荐时，是否应显式拆成:
  - 学习入口推荐
  - 直接试用推荐
  - 长期跟踪推荐
