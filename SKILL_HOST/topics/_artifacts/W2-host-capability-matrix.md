## Early decision guidance

- If the goal is `highest workflow sophistication`, Claude currently leads.
- If the goal is `engineered runtime clarity and explicit controls`, Codex currently leads.
- If the goal is `IDE-native layering and rapid multi-environment expansion`, Cursor is the key host to watch.
- If the goal is `compatibility, bridging, and explicit tool/provider control`, OpenCode is the strongest current sample.
- If the goal is `predictable explicit constraints`, prefer hosts whose runtime assumptions are documented rather than partly hidden behind backend routing.

## Caveat

- This matrix compares current evidence-backed tendencies, not immutable truths.
- The same skill can appear compatible at `format` level while still failing or degrading at `runtime` level.
- For advanced skills, `execution topology` and `constraint visibility` are now as important as file-format compatibility.
