# Build 阶段

## 这一阶段在解决什么

Build 阶段不是单纯“开始写代码”，而是回答五个问题：

- 该按什么节奏写
- 证据从哪里来
- agent 需要哪些上下文
- UI 怎么避免 AI 味
- API 和模块边界怎么避免后患

这个阶段的 skill 数量最多，也说明仓库作者认为：真正容易失控的，就是实现阶段。

## 1. incremental-implementation

### 职责

- 要求按薄切片实现
- 每个切片都要测试、验证、提交
- 防止一口气大改

### 我的理解

这是“执行纪律 skill”。  
对 AI agent 特别重要，因为 agent 一旦开始写，很容易生成过量代码。

### 优点

- 每步都可回退
- 降低 diff 爆炸风险
- 适合 feature flag 与分阶段落地

### 缺点

- 依赖测试与提交流程
- 原型场景里会显慢

### 判断

- 工程实用性：非常高
- AI 友好度：非常高
- 推荐优先级：最高级别之一

## 2. test-driven-development

### 职责

- 先写失败测试，再写实现
- 修 bug 时先复现再修
- 最终用测试证明结果

### 我的理解

这是最强的“反自我感觉良好” skill。  
AI 最容易在这里出问题：以为修好了，其实没有。

### 优点

- 回归保护强
- bug fix 特别有价值
- 能显著提升变更可信度

### 缺点

- 探索型任务会有摩擦
- 测试基础薄弱时成本上升

### 判断

- 工程实用性：非常高
- AI 友好度：非常高
- 推荐优先级：最高级别之一

## 3. context-engineering

### 职责

- 决定 agent 应该看到什么上下文
- 决定规则、spec、源码、错误信息的装载顺序
- 降低跑偏和幻觉

### 我的理解

这个 skill 的本质是“喂上下文的工程学”。  
不是模型不够聪明，而是上下文质量常常不够高。

### 优点

- 非常适合大仓库
- 对长期 session 很有帮助
- 对项目约定沉淀价值高

### 缺点

- 需要持续维护 rules
- 上下文打包本身也需要经验

### 判断

- 工程实用性：高
- AI 友好度：极高
- 推荐优先级：高

## 4. frontend-ui-engineering

### 职责

- 产出真实产品级 UI
- 保证组件边界、状态管理、设计系统一致、可访问性达标
- 避免生成式 UI 的常见套路味

### 我的理解

它其实不是一个普通前端规范，而是一个“反 AI 风格污染”的 skill。  
这点很聪明，因为现在很多 agent 产出的 UI 第一眼就像 AI 生成品。

### 优点

- 对前端质量提升非常明显
- 把 a11y 放进工程流程
- 对组件职责划分也有帮助

### 缺点

- 偏 web 语境
- 没有设计系统时会弱化
- 一部分判断带审美经验成分

### 判断

- 工程实用性：中高
- AI 友好度：高
- 推荐优先级：前端项目高，后端项目低

## 5. api-and-interface-design

### 职责

- 设计稳定接口
- 用契约先行约束实现
- 避免未来因为边界混乱导致的大量返工

### 我的理解

这是“Build 阶段里最偏长期主义”的 skill。  
它关心的不是今天能不能跑，而是三个月后还能不能改。

### 优点

- 有利于模块边界清晰
- 有利于跨团队协作
- Hyrum's Law 和 One-Version Rule 都非常实用

### 缺点

- 对内部一次性脚本偏重
- 需要团队认同统一错误语义

### 判断

- 工程实用性：高
- AI 友好度：高
- 推荐优先级：涉及接口时极高

## Build 阶段的组合建议

### 普通功能开发

- `context-engineering`
- `incremental-implementation`
- `test-driven-development`

### 前端功能开发

- 上面的组合
- 再加 `frontend-ui-engineering`

### API 或模块设计

- 上面的组合
- 再加 `api-and-interface-design`

## Build 阶段常见失败模式

- 没有上下文就开始写
- 没有切片，一次改太多文件
- 没有先补测试就直接改行为
- UI 只管长出来，不管一致性与可访问性
- API 只追求能通，不考虑演进

## 我的总评

Build 阶段是这套仓库最“防 agent 失控”的部分。  
如果只让我挑最值得长期保留的 skill，Build 阶段至少能占一半。
