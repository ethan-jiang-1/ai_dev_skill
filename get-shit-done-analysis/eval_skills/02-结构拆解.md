# get-shit-done 结构拆解

## 结构判断

从“skill-like 内容”这个视角看，  
这个仓库可以拆成五层：

1. 入口层：`commands/gsd/*.md`
2. 角色层：`agents/*.md`
3. 编排层：`get-shit-done/workflows/*.md`
4. 规则层：`get-shit-done/references/*.md`
5. 工件层：`get-shit-done/templates/*.md`

外围还有：

- `bin/`：安装与工具脚本
- `docs/`：解释和文档
- `tests/`：回归验证

但核心仍是上面这五层。

## 一、入口层：commands

这一层最像“显式 skill 注册表”。

典型结构：

- front matter
- objective
- execution_context
- process

以 `commands/gsd/new-project.md` 为例，可以看到：

- `name: gsd:new-project`
- `description: ...`
- `allowed-tools: ...`
- `execution_context` 指向 workflow 和 references

这意味着 command 文件承担三件事：

- 对 runtime 暴露一个能力入口
- 限定该入口可以使用的工具
- 把真正执行逻辑路由到 workflow

## 二、角色层：agents

这一层最像“专职子技能”。

agent front matter 不是装饰，  
而是能力声明的一部分。

典型字段：

- `name`
- `description`
- `tools`
- `color`

正文再补足：

- role
- core responsibilities
- project context
- fidelity rules
- prohibition rules

也就是说，agent 文件本质上是在声明一个有边界的执行角色。

## 三、编排层：workflows

workflow 是整套系统最像“变形 skill”的部分。

它没有统一叫 skill，  
但它确实承担了 skill orchestration 的职责：

- 读取 invoking prompt 的上下文
- 解析参数
- 决定是否提问
- 调用 agent
- 写入 `.planning/` 状态
- 推进到下一阶段

这层的关键价值是把原本会散在 prompt 里的流程变成可维护的结构化脚本。

## 四、规则层：references

references 让 skill 系统变得稳定。

如果没有这一层：

- 每个 command 会重复写规则
- 每个 agent 会重复写原则
- 不同阶段会逐渐产生口径漂移

有了 references 之后：

- questioning
- verification-patterns
- context-budget
- planning-config
- tdd

这些规则就变成可复用的共享约束。

## 五、工件层：templates

templates 让 system output 也被结构化。

它们服务的是 `.planning/` 体系：

- `PROJECT.md`
- `REQUIREMENTS.md`
- `ROADMAP.md`
- `STATE.md`
- `VALIDATION.md`
- `UAT.md`

这意味着这个仓库不只在定义“怎么做”，  
也在定义“做完后留下些什么”。

## 结构上的关键洞察

### 1. command 和 agent 有显式 front matter

这是最直接的 skill 识别信号。

它们已经具备很多 skill catalog 会有的元数据特征：

- 名称
- 描述
- 工具权限
- 角色定位

### 2. workflow 是非 front matter 但强结构的 skill

它虽然长得不像标准 skill 卡片，  
但它是实际执行路径里的主心骨。

### 3. reference 和 template 是技能系统的基础设施

如果只看 front matter，容易低估它们。  
但从系统能力角度看，它们是 skill 能稳定复用的原因。

## 结构结论

`get-shit-done` 不是“很多 markdown 文件堆在一起”。  
它是一个有明确分层的 skill-like 系统：

- command 负责暴露入口
- agent 负责承接专职执行
- workflow 负责编排
- reference 负责统一规则
- template 负责统一产物

这五层一起，构成了它真正可复用的部分。
